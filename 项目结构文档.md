## 二、核心文件说明

### 2.1 入口文件

#### 📄 `src/main.js` - 应用主入口
**功能：** Vue应用初始化入口
**关联：** 引入 `App.vue` 组件

#### 📄 `src/App.vue` - 应用根组件
**功能：** 应用生命周期管理
**关联：** 无直接关联，所有页面通过 `pages.json` 配置

#### 📄 `src/pages.json` - 页面路由配置
**功能：** 定义所有页面路径和导航栏配置
**关联：** 引用所有 `pages/` 下的页面组件
```json
{
  "pages": [
    "pages/index/index",      // 首页
    "pages/me/me",            // 我的
    "pages/detail/detail",    // 详情
    "pages/apply/apply",      // 入驻申请
    "pages/audit-status/audit-status",  // 审核状态
    "pages/profile/profile",  // 个人信息
    "pages/fav/fav",          // 我的收藏
    "pages/rules/rules",      // 平台规则
    "pages/rule-detail/rule-detail",  // 规则详情
    "pages/register/register" // 注册登录
  ]
}
```

#### 📄 `src/manifest.json` - 应用配置清单
**功能：** 小程序配置（AppID、权限等）
**关联：** 无直接关联

#### 📄 `src/uni.scss` - 全局样式变量
**功能：** 定义全局颜色、字体等样式变量
**关联：** 所有 `.vue` 文件通过 `@import '@/uni.scss'` 引入

---

### 2.2 状态管理

#### 📄 `src/store/app.js` - 全局状态管理
**功能：** 管理应用全局状态
**导出内容：**
```javascript
// 状态对象
export const appState = reactive({
  likedIds: [],              // 收藏列表
  auditStatusValue: '',      // 审核状态
  meAvatar: '',              // 用户头像
  meName: '',                // 用户昵称
  orderEnabled: false,       // 接单状态
  currentDetailCard: null,   // 当前详情卡片
  isLoggedIn: false          // 登录状态
})

// 状态管理函数
export function isLiked(id)           // 判断是否收藏
export function toggleLike(id)        // 切换收藏状态
export function setAuditStatus(val)   // 设置审核状态
export function setMeProfile(obj)     // 设置个人信息
export function setLoggedIn()         // 设置登录状态
```

**被引用关系：**
- ✅ `pages/index/index.vue` - 首页收藏功能
- ✅ `pages/detail/detail.vue` - 详情页收藏功能
- ✅ `pages/me/me.vue` - 我的页面用户信息
- ✅ `pages/fav/fav.vue` - 收藏列表
- ✅ `pages/profile/profile.vue` - 个人信息编辑
- ✅ `pages/register/register.vue` - 登录状态设置
- ✅ `pages/apply/apply.vue` - 入驻申请
- ✅ `pages/audit-status/audit-status.vue` - 审核状态

## 三、页面文件详解

### 3.1 首页模块

#### 📄 `src/pages/index/index.vue` - 首页
**功能：** 摄影师列表展示与筛选

**页面结构：**
```
首页
├── 顶部标题区
│   ├── 标题：校里·约拍
│   └── 学校标签：广西医科大学
├── 标签切换区
│   ├── 新入驻
│   ├── 接单中
│   └── 筛选按钮（带徽章）
├── 摄影师列表区
│   └── 卡片组件（循环）
│       ├── 头像 + 昵称 + 认证图标
│       ├── 副标题 + 价格
│       ├── 个人简介
│       ├── 作品展示（图片网格）
│       └── 底部标签 + 收藏按钮
├── 筛选弹层
│   ├── 擅长风格选择（11个标签）
│   ├── 价格区间滑块（自定义拖拽，0-9999）
│   ├── 自定义输入框
│   ├── 重置按钮
│   └── 完成按钮
└── 底部导航
    ├── 约拍（激活）
    └── 我的
```

**关键特性（最新更新）：**
- ✅ 自定义拖拽滑块实现（替代 uni-slider），支持 H5 和小程序
- ✅ 滚动条自动显隐功能（滚动时显示，停止0.8秒后淡出）
- ✅ 价格上限从 588 提升至 9999
- ✅ 筛选风格标签增加"女摄"、"同校"

**数据流向：**
```
data.js (cardsList, filterStyles, PRICE_CAP)
    ↓
index.vue (filteredCards)
    ↓
渲染卡片列表
    ↓
用户点击收藏
    ↓
store/app.js (toggleLike)
    ↓
更新 appState.likedIds
```

**引用关系：**
- 🔗 引入 `@/utils/data.js` - 获取摄影师列表、筛选选项、PRICE_CAP
- 🔗 引入 `@/store/app.js` - 收藏功能、全局状态
- 🔗 引入 `@/utils/system.js` - 获取状态栏高度
- 🔗 引入 `@/uni.scss` - 样式变量

**跳转关系：**
- ➡️ 点击卡片 → `pages/detail/detail?id={id}`
- ➡️ 点击收藏 → 触发登录弹窗（未登录时）
- ➡️ 点击"我的" → `pages/me/me`

```
我的页面

│   ├── 平台规则 → 跳转 rules

- ✅ 未登录状态显示优化（灰色占位头像 + 登录提示）

**数据流向：**
```
store/app.js (appState)
    ↓

    ↓

    ↓

    ↓

    ↓
requireLogin 守卫检查
    ↓
已登录 → 跳转对应页面
未登录 → 弹出登录弹窗
```

### 5.2 状态更新流程

#### 收藏功能流程
```
用户点击收藏按钮
    ↓
调用 toggleLike(id)
    ↓
检查 likedIds 是否包含该ID
    ├─ 已包含 → 移除（splice）
    └─ 未包含 → 添加
    ↓
更新 appState.likedIds
    ↓
自动触发UI更新（响应式）
```

#### 登录流程
```
用户点击登录
    ↓
弹出登录弹窗
    ↓
微信授权确认
    ↓
调用 setLoggedIn()
    ↓
appState.isLoggedIn = true
    ↓
自动触发UI更新
    ↓
显示用户信息
```

#### 审核状态流程
```
用户提交入驻申请
    ↓
调用 setAuditStatus('pending')
    ↓
appState.auditStatusValue = 'pending'
    ↓
跳转到审核页
    ↓
审核页读取状态显示对应界面
```

#### 登录守卫流程（requireLogin）
```
用户点击需要登录的功能
    ↓

    ↓

    ├─ 未登录 → 弹出登录框，阻断操作

## 八、开发建议

### 8.1 新增页面流程

1. 在 `src/pages/` 下创建新的页面文件夹和 `.vue` 文件
2. 在 `src/pages.json` 中添加页面配置
3. 在 `src/uni.scss` 中定义页面所需样式变量（如有）
4. 在 `src/store/app.js` 中添加相关状态（如有）
5. 在 `src/utils/data.js` 中添加数据（如有）

### 8.2 状态管理建议

- **局部状态：** 使用 `ref()` 或 `reactive()` 在组件内部管理
- **跨页面状态：** 使用 `appState` 全局管理
- **状态更新：** 通过导出的函数更新，避免直接修改

### 8.3 数据管理建议

- **静态数据：** 放在 `src/utils/data.js`
- **动态数据：** 后期对接API后，替换为接口调用
- **数据格式：** 保持与 `data.js` 中的数据结构一致
- **常量统一：** 价格上限等常量使用 `PRICE_CAP`，避免硬编码

### 8.4 风格标签统一

- **首页筛选** 和 **入驻申请** 的擅长风格标签都从 `data.js` 的 `filterStyles` 引入
- **当前标签列表：** 人像摄影、胶片复古、日系清新、毕业写真、情绪片、互勉、证件照、棚拍、日系、女摄、同校
- **如需修改标签：** 只需修改 `data.js` 中的 `filterStyles` 数组
